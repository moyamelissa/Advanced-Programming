La programmation concurrente en Go repose sur les goroutines et les canaux.
Les goroutines sont des fils d'exécution légers gérés par le runtime de Go.
Les canaux permettent la communication sûre entre les goroutines sans partage de mémoire.
Ce modèle est inspiré du CSP Communicating Sequential Processes de Tony Hoare.

Un programme bien conçu utilise la concurrence pour diviser le travail en tâches indépendantes.
Chaque tâche peut être exécutée en parallèle sur les cœurs disponibles du processeur.
Le canal sert de point de synchronisation garantissant que les résultats sont collectés correctement.
Il est important de fermer les canaux quand ils ne sont plus nécessaires pour éviter les fuites de goroutines.
